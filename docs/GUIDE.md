## 📊 실전 거래 프로세스

### 1️⃣ 진입 시점 (Entry)
```python
# 명령어 실행
python run.py

# 확인 사항:
✅ 15분 신호: LONG/SHORT (신뢰도 70%+)
✅ 30분 신호: 방향 확인
✅ 4시간 트렌드: 같은 방향
✅ 1일 바이어스: 전체 시장 방향
```

**타임프레임 정렬 전략:**
- 4/4 정렬: 최대 포지션 (자본의 5%)
- 3/4 정렬: 정상 포지션 (자본의 3%)
- 2/4 정렬: 소량 포지션 (자본의 1%)
- 1/4 이하: 거래 금지

### 2️⃣ 포지션 관리 (Management)
```
⏰ 15분마다: 재분석 요청
   → 신호 유지 확인
   → 신뢰도 변화 체크

⏰ 30분마다: 추가 확인 (선택)
   → 중기 방향성 확인

⏰ 1시간마다: 트렌드 체크
   → 4시간 트렌드와 일치 확인
```

### 3️⃣ 청산 시점 (Exit)
```
즉시 청산:
❌ 15분 신호 반전
❌ 신뢰도 < 65%
❌ 4시간 트렌드 반전

목표 청산:
✅ 수익률 +3% 도달
✅ 4시간 경과 (장기 보유 금지)
✅ 손실 -2% 도달 (손절)
```

## 🔧 실행 명령어

### 실시간 신호 확인
```bash
# 통합 시스템 (모든 모델)
python run.py

# 또는 MCP 서버 실행 (Claude Desktop)
python mcp_server.py

# 출력 예시:
# 15분: LONG (81.5%)
# 30분: LONG (72.1%)
# 4시간: LONG (78.6%)
# 1일: NEUTRAL (75.0%)
# 종합 신호: LONG
# 타임프레임 정렬: 3/4 (75%)
# 추천 포지션: 자본의 3%
```

### 신호 해석
```python
신뢰도 >= 70%: "강한 신호" → 정식 포지션
신뢰도 65-70%: "보통 신호" → 소량/주의
신뢰도 < 65%: "약한 신호" → 거래 금지
```

```
항상 -2% 손절
절대 손절선 제거 금지
```

```
4시간 이내 청산
```
